# Security Policy

## Reporting a Vulnerability

Please do not open a public GitHub issue for vulnerabilities, leaked credentials, private
operational details, or sensitive logs.

Report security concerns privately through the repository owner's GitHub profile. Include enough
detail to reproduce or evaluate the issue, such as affected versions, expected impact, and a
minimal reproduction when one is safe to share.

## Sensitive Information

Never commit API keys, tokens, private credentials, provider response bodies, prompts, user data, or
other sensitive values to this repository. Redact sensitive values from logs, test fixtures, issue
reports, and pull request descriptions before sharing them.

If a secret is accidentally exposed, rotate it immediately with the issuing provider before
removing it from the repository history or public discussion.
